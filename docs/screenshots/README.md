이 문서는 분양 기능 관련 스크린샷을 보기 쉽게 정리한 안내 자료입니다. 상위 README에서는 대표적인 핵심 화면만 간단히 소개하지만, 여기에서는 실제 서비스 안에서의 전체 화면 흐름을 하나씩 세밀하게 살펴볼 수 있습니다.

> 참고로, 아래에 소개된 화면 중 일부는 분양 기능 자체의 디테일보다는, 분양 기능이 서비스 내에서 어떤 위치와 맥락에 배치되어 있는지 보여주기 위해 함께 정리한 사례입니다.

## 추천되는 읽는 순서

1. 리스트와 상세
2. 신청자 관리 및 완료
3. 마이페이지, 알림
4. 후기 작성 플로우
5. 서비스 맥락 화면

---

## 01. 분양 리스트

![분양 리스트](01_adoption_list.png)

### 화면 설명
분양 글들을 카드 형태로 한눈에 살펴볼 수 있는 화면입니다. 시각적으로 구분되어 어떤 글이 지금 어떤 상태에 있는지 빠르게 파악할 수 있습니다.

### 주요 기능
- 상태별 카드 구분(모집중, 진행중, 완료)
- 검색, 필터, 정렬 기능 제공
- 목록에서 현재 분양 진행 상태 바로 확인

### 담당한 부분
상세 페이지로 따로 들어가지 않아도, 리스트에서 모집중/진행중/완료 상태를 즉시 알 수 있도록 설계했습니다. 사용자가 여러 글 중 필요한 정보를 바로 찾을 수 있게 하는 점에 중점을 두었습니다.

---

## 02. 분양 상세(모집중)

![분양 상세](02_adoption_detail_open.png)

### 화면 설명
모집중 상태의 분양 상세 페이지입니다. 신청 및 각종 버튼이 적절히 노출됩니다.

### 주요 기능
- 신청 버튼 표시
- 현재 신청 내용 확인
- 작성자와 방문자에 따라 버튼 구분 노출

### 담당한 부분
상세 화면에서는 사용자의 로그인 여부와 글의 상태에 따라 신청/취소/승인/완료 등 버튼이 다르게 노출됩니다. 이를 위해 상태와 권한을 우선 체크하고, 그에 맞춰 화면을 유연하게 구성했습니다.

---

## 03. 신청자 관리

![신청자 관리](03_owner_manage_applicants.png)

### 화면 설명
글 작성자가 신청자 목록을 보고 승인 또는 거절을 처리할 수 있는 관리용 화면입니다.

### 주요 기능
- 신청자 목록 확인
- 승인/거절 버튼
- 현재 분양 진행 상태 실시간 확인

### 담당한 부분
가장 중요하게 고려한 것은 **최종 승인자는 반드시 한 명만 남도록 한다는 점**이었습니다. 화면에서 승인 버튼이 여러 번 눌려도, 서버에서 한 번 더 검증하여 중복된 승인이 발생하지 않도록 안전장치를 배치했습니다.

---

## 04. 분양 완료

![분양 완료](04_adoption_completed.png)

### 화면 설명
승인 절차가 마무리되고 분양이 실제로 완료된 직후의 화면입니다. 종료된 게시글의 상태와 연동됩니다.

### 주요 기능
- 완료된 상태 표시
- 종료된 분양글 UI 반영
- 후기 작성으로 바로 이어지는 흐름

### 담당한 부분
분양 완료 처리는 단순히 화면에 ‘완료’라고 표기하는 것을 넘어, 실제 동물 소유권이 이전되고 게시글도 종료로 바뀌어야 합니다. 데이터의 실질적 변화와 서비스 흐름이 자연스럽게 이어지도록, 모든 화면 요소를 한 프로세스로 연결했습니다.

---

## 05. 마이페이지 신청 내역

![마이페이지 신청 내역](05_mypage_applies.png)

### 화면 설명
내가 신청한 분양 내역을 한눈에 확인할 수 있는 마이페이지 화면입니다.

### 주요 기능
- 신청 글 목록 나열
- 각 글의 현재 상태 표시
- 분양 후 단계별 진행 상황 추적

### 담당한 부분
분양 기능은 단순 게시판의 일회성 기능이 아니라, 신청자가 마이페이지에서 자신의 진척 상황을 직접 점검하는 루트까지 고려해야 했습니다. 실제 사용자의 서비스 동선을 따라 게시판-마이페이지 흐름을 자연스럽게 연결했습니다.

---

## 06. 마이페이지 포인트

![마이페이지 포인트](06_mypage_points.png)

### 화면 설명
회원이 자신의 포인트를 확인할 수 있는 마이페이지 내 부가 화면입니다.

### 참고한 이유
이 스크린샷은 PETIQUE 전체 마이페이지 구성의 맥락을 보여주는 보충 자료입니다. 분양 관련 기능도 마이페이지에서 연동되기에, 서비스의 전체 흐름과 사용자 경험을 설명하는 데 활용했습니다.
이처럼 각 화면은 개별 기능에만 집중하지 않고, 실제 PETIQUE 서비스를 처음부터 끝까지 자연스럽게 이어지는 사용자 시각에서 설계·구성했습니다. 서비스 전반의 맥락을 충분히 고려하여 상세 설명을 준비했습니다.

## 07. 회원 프로필

![회원 프로필](07_member_profile.png)

## 09. 후기 연결 화면

![후기 연결](09_adoption_detail_review.png)

### 화면 설명
분양이 완료된 후, 상세 화면에서 바로 후기 작성 또는 보기가 이어질 수 있도록 동선을 연결한 화면입니다.

### 여기서 보이는 기능
- 신청이 완료된 회원에게만 후기 작성 버튼 제공
- 이미 후기가 있는 경우, 해당 글로 바로 이동

### 이 화면에서 담당한 역할
분양이 마무리된 이후에도 사용자가 자연스럽게 후기 작성까지 이동할 수 있도록 동선을 설계했습니다. 이를 통해 사용 경험이 끊기지 않도록 신경 썼습니다.

## 10. 후기 작성

![후기 작성](10_review_write.png)

### 화면 설명
실제로 분양 후기를 작성하는 공간입니다.

### 여기서 보이는 기능
- 분양 후기 작성
- 기존 분양글과 후기글을 자연스럽게 연결
- 중복 후기 작성 방지

### 이 화면에서 담당한 역할
분양이 완료된 신청자만 후기 작성 폼에 접근할 수 있도록 했고, 이미 작성된 후기가 있다면 기존 글로 안내되도록 설계했습니다.

---

## 11. 마이페이지 알림

![마이페이지 알림](11_mypage_notifications.png)

### 화면 설명
신청부터 승인, 거절, 그리고 분양 완료 후까지 발생하는 다양한 알림을 모아 한 번에 확인할 수 있는 화면입니다.

### 여기서 보이는 기능
- 각 단계별 상태 변화 알림 제공
- 알림을 통해 필요한 다음 단계로 바로 이동 가능

### 이 화면에서 담당한 역할
분양 과정은 신청 이후 단계별로 다양한 변화가 발생합니다. 사용자가 중요한 알림을 놓치지 않도록 마이페이지 내 알림 동선을 마련하여, 분양 게시판 기능이 단순 게시글로만 끝나지 않도록 했습니다.

---

## 짧게 정리하면

분양 게시판은 ‘신청-승인-완료-후기-알림’까지 하나의 유기적인 흐름으로 이어집니다. 각 단계가 어떻게 연결되는지 사용자가 쉽게 이해할 수 있도록, 실제 프로세스 순서에 맞게 스크린샷을 그룹화해 정리했습니다. 이러한 구성 덕분에 서비스 전체의 맥락 속에서 분양 기능의 흐름을 명확히 보여줄 수 있었습니다.
