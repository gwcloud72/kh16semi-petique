# PETIQUE | 화면 상세 설명

이 문서는 분양 기능과 관련된 스크린샷을 한눈에 보기 쉽게 정리한 페이지입니다. 상위 README 문서에서는 주요 대표 화면만 간략히 소개하며, 본 문서에서는 전체 화면 흐름을 상세하게 확인할 수 있습니다.

> 참고로 아래 화면 중 일부는 분양 기능 자체보다, 분양 기능이 서비스 안에서 어디에 놓여 있는지 보여주기 위해 함께 남겨둔 화면입니다.

## 읽는 순서

1. 리스트와 상세
2. 신청자 관리와 완료
3. 마이페이지와 알림
4. 후기 작성 흐름
5. 서비스 맥락 화면

---

## 01. 분양 리스트

![분양 리스트](01_adoption_list.png)

### 화면 설명
분양글을 카드형으로 확인하는 화면입니다.

### 여기서 보이는 기능
- 상태별 카드 구분
- 검색 / 필터 / 정렬
- 목록에서 현재 분양 진행 상태 확인

### 이 화면에서 담당한 기능
리스트에서도 모집중 / 진행중 / 완료 상태가 바로 보이게 정리했습니다. 상세에 들어가야만 상태를 아는 구조보다, 목록에서 현재 글 상태를 빠르게 읽을 수 있게 맞추는 것이 중요했습니다.

---

## 02. 분양 상세(모집중)

![분양 상세](02_adoption_detail_open.png)

### 화면 설명
모집중 상태의 상세 화면입니다.

### 여기서 보이는 기능
- 신청 버튼 노출
- 신청 내용 확인
- 작성자 / 방문자 기준 버튼 구분

### 이 화면에서 담당한 기능
상세 화면에서는 신청 / 취소 / 승인 / 완료 버튼이 로그인 사용자와 글 상태에 따라 다르게 보여야 했습니다. 그래서 상태와 권한을 먼저 계산한 뒤 화면에서 표시만 하게 정리했습니다.

---

## 03. 신청자 관리

![신청자 관리](03_owner_manage_applicants.png)

### 화면 설명
작성자가 신청자 목록을 보고 승인 / 거절을 처리하는 화면입니다.

### 여기서 보이는 기능
- 신청자 목록 조회
- 승인 / 거절 버튼
- 현재 분양 진행 상태 확인

### 이 화면에서 담당한 기능
분양에서 가장 중요했던 건 **승인자가 한 명만 남아야 한다**는 점이었습니다. 화면에서 버튼을 눌러도 서버에서 다시 한 번 검증해 중복 승인 상태가 생기지 않도록 맞췄습니다.

---

## 04. 분양 완료

![분양 완료](04_adoption_completed.png)

### 화면 설명
승인 이후 실제로 분양이 완료된 뒤의 화면입니다.

### 여기서 보이는 기능
- 완료 상태 표시
- 종료된 분양글 UI 반영
- 후기 작성 흐름 진입 준비

### 이 화면에서 담당한 기능
분양 완료 처리는 단순한 상태값 변경을 넘어, 실질적인 동물 소유권 이전과 분양 게시글의 종료 상태가 동시에 반영되어야 합니다. 이러한 실제 데이터의 변화와 상태 동기화 과정이 직관적으로 드러나도록 관련 화면들을 하나의 프로세스로 묶어 구성했습니다.

---

## 05. 마이페이지 신청 내역

![마이페이지 신청 내역](05_mypage_applies.png)

### 화면 설명
내가 신청한 분양 내역을 확인하는 마이페이지 화면입니다.

### 여기서 보이는 기능
- 신청한 글 목록 확인
- 현재 상태 확인
- 분양 이후 흐름 추적

### 이 화면에서 담당한 기능
분양 기능은 게시판 내의 단편적인 동작에 그치지 않고, 신청자가 마이페이지에서 본인의 진행 상태를 조회하는 과정까지 포함합니다. 이러한 실제 사용자의 서비스 이용 동선을 반영하여, 게시판에서 마이페이지로 이어지는 화면 흐름을 순서대로 구성했습니다.

## 06. 마이페이지 포인트

![마이페이지 포인트](06_mypage_points.png)

### 화면 설명
회원 포인트를 확인하는 마이페이지 화면입니다.

### 참고 이유
PETIQUE 전체 마이페이지의 구성을 보여주는 보조 스크린샷입니다. 분양 관련 기능 또한 해당 마이페이지 내에서 연결되어 동작하므로, 서비스의 전체적인 흐름과 문맥을 설명하기 위해 함께 포함했습니다.

## 07. 회원 프로필

![회원 프로필](07_member_profile.png)

### 화면 설명
회원과 반려동물 정보를 보는 화면입니다.

## 09. 후기 연결 화면

![후기 연결](09_adoption_detail_review.png)

### 화면 설명
분양 완료 뒤 상세 화면에서 후기 작성 / 보기 흐름이 연결된 상태입니다.

### 여기서 보이는 기능
- 완료된 신청자에게만 후기 버튼 노출
- 이미 후기글이 있으면 보기로 연결

### 이 화면에서 담당한 기능
분양이 완료되면 그다음 행동이 끊기지 않게, 후기 작성 동선까지 자연스럽게 이어지도록 붙였습니다.

---

## 10. 후기 작성

![후기 작성](10_review_write.png)

### 화면 설명
분양 후기 글을 실제로 작성하는 화면입니다.

### 여기서 보이는 기능
- 분양 후기 작성
- 기존 분양글과 후기글 연결
- 중복 작성 방지

### 이 화면에서 담당한 기능
완료된 신청자만 후기 쓰기 폼에 들어오게 하고, 이미 연결된 후기글이 있으면 다시 쓰지 않고 기존 글로 보내도록 정리했습니다.

---

## 11. 마이페이지 알림

![마이페이지 알림](11_mypage_notifications.png)

### 화면 설명
신청 / 승인 / 거절 / 완료 이후 알림을 확인하는 화면입니다.

### 여기서 보이는 기능
- 상태 변화 알림 확인
- 다음 행동으로 이동할 수 있는 링크 제공

### 이 화면에서 담당한 기능
분양 기능은 신청 이후 상태 변화가 많아서, 사용자가 그 변화를 놓치지 않도록 알림 동선을 같이 붙였습니다. 게시판 안에서만 끝나는 기능으로 보이지 않게 하는 데 도움이 된 화면입니다.

---

## 짧게 정리하면

분양 게시판은 '신청-승인-완료-후기-알림'으로 이어지는 하나의 긴밀한 프로세스입니다. 이 흐름을 사용자가 한눈에 이해할 수 있도록, 각 단계의 상태 변화 순서에 따라 스크린샷을 그룹화하여 정리했습니다.
