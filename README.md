# 🐾 PETIQUE — 반려동물 정보 공유 및 소통 커뮤니티

PETIQUE는 반려동물 관련 정보를 공유하고, 사용자 간 소통과 커뮤니티 활동을 지원하는 **회원제 기반 반려동물 커뮤니티 서비스**입니다.  
회원 관리, 게시판/댓글, 쪽지, 포인트, 마이페이지, 반려동물 관리 기능을 기반으로 **분양 정보 공유와 분양 진행 상태 관리**까지 연결되는 구조로 기획했습니다.

본 프로젝트는 **팀 프로젝트**로 진행되었으며, 저는 그중 **분양 게시판 영역과 분양 프로세스 구현**을 담당했습니다.  
단순히 분양 글을 등록하는 수준이 아니라, **신청 → 승인 → 거절 → 완료**로 이어지는 흐름을 설계하고, 해당 상태가 DB·권한·UI에 일관되게 반영되도록 구현한 점이 핵심입니다.

---

## 📌 프로젝트 소개

- 반려동물 관련 정보 공유 및 사용자 간 소통을 위한 커뮤니티 서비스
- 회원제 기반으로 신뢰성 있는 정보 교류 환경 제공
- 게시판/댓글/쪽지/포인트/마이페이지 기능 제공
- 반려동물 등록 및 프로필 관리 기능 제공
- 분양 게시판을 통해 분양 가능 여부, 문의 접수, 진행 상태 관리 지원

## 🙋 담당 역할

팀 프로젝트 내에서 다음 영역을 담당했습니다.

- **`ADOPTION_APPLY`, `BOARD_ANIMAL`, `ANIMAL` 연동 로직 구성**
- **상태 기반 UI 제어 및 권한 처리**
- **마이페이지 신청 내역, 알림, 후기 작성 동선 등 관련 기능 연동**

## ✅ 담당 기능 구현

### 1. 분양 게시판 UI/UX
- 동물 분류, 게시판 타입 기반 필터링
- 검색 기능 및 최신순/조회순 정렬
- 12개 단위 페이지네이션
- 리스트/상세에서 분양 가능 여부와 진행 상태 표시

### 2. 분양 프로세스 상태 관리
- 신청 내용 및 처리 이력을 `ADOPTION_APPLY`에 기록
- 작성자는 **1명만 승인 가능**하도록 처리
- 승인 후 남은 대기 신청은 자동 정리(거절 처리)
- 상태에 따라 신청/취소/승인/거절/완료 버튼을 분기

### 3. 완료 처리와 데이터 반영
- 승인된 신청을 `COMPLETED` 상태로 변경
- 동물 소유자(`ANIMAL_MASTER`)를 승인자에게 이전
- `ANIMAL_PERMISSION = 'f'`로 전환해 분양 종료 상태를 반영
- 리스트/상세에서 종료 상태가 즉시 보이도록 처리

### 4. 연동 기능
- 마이페이지에서 내가 신청한 분양 내역 확인
- 분양 신청/승인/거절/완료 시 알림 저장 및 확인
- 분양 완료 후 후기 작성/보기 동선 연결

👉 분양 프로세스의 설계 의도, 상태 정의, 핵심 컨트롤러/서비스/DAO/쿼리, 트러블슈팅은 `detail.md`에 정리했습니다.

## 🧩 프로젝트 주요 기능

### 회원 관리
- 회원가입 / 로그인 / 로그아웃
- 이메일 인증을 통한 가입 승인
- 회원 정보 수정 및 마이페이지 관리

### 게시판 / 댓글
- 게시글 작성 / 수정 / 삭제 / 조회
- 카테고리별 게시판 운영
- 댓글 등록 및 관리
- 조회수 / 추천수 관리

### 쪽지 / 마이페이지
- 회원 간 쪽지 송수신
- 작성글 모아보기 및 삭제
- 포인트 내역 및 활동 기록 확인

### 포인트 / 회원 등급
- 게시글 작성 및 활동 기반 포인트 적립
- 삭제/차감 정책 반영
- 회원 등급 기준 및 등급별 관리

### 반려동물 관리
- 반려동물 등록 및 소개글 관리
- 프로필 이미지 등록/수정
- 분양 가능 여부(permission flag) 관리

### 분양 게시판
- 분양글 등록 및 조건 설정
- 분양 문의 접수 시스템
- 분양 상태 업데이트(모집중 / 진행중 / 완료)
- 사용자 친화적 필터/정렬 UI 제공

## 🏗 Architecture

| System Architecture |
|---|
| ![](docs/diagrams/architecture.png) |

프로젝트는 MVC 구조를 기반으로 구성했습니다.

요청 흐름은  
**Controller → Service → DAO → Database**  
순서로 동작합니다.

특히 분양 승인 및 완료 처리처럼 여러 테이블이 함께 변경되는 작업은  
Service 계층에서 트랜잭션으로 묶어 데이터 일관성을 유지했습니다.
