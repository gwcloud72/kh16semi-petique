# PETIQUE | 분양 게시판

PETIQUE는 반려동물 정보를 기록하고, 커뮤니티와 분양 기능까지 결합한 팀 프로젝트입니다.

저는 그중에서도 분양 게시판 파트를 담당했습니다. 겉으로만 보면 단순한 게시판 같지만, 실제로는 '신청-승인-완료-후기'로 단계가 이어지는 특성상, 게시판이라기보다는 하나의 프로세스로 접근해 설계하고 구현했습니다.

---

## 프로젝트 소개

- 반려동물 정보부터 커뮤니티 요소, 그리고 분양까지 아우르는 서비스입니다.
- 분양 게시글에서 시작해, 신청 내역 관리, 각 단계별 알림, 후기까지 한 흐름으로 연결합니다.
- 작성자, 신청자, 일반 방문자에 따라 화면 동선과 보여지는 상태가 다르게 구성되도록 설계되었습니다.

---

## 담당 역할

- 분양 리스트 및 상세 화면, 신청자 관리 화면 개발
- 분양 신청, 취소, 승인, 거절, 완료 등 전반적인 프로세스 설계 및 구현
- 상태별 버튼 노출과 각 역할별 권한 처리 세분화
- 분양 후기, 각 단계별 알림, 마이페이지와의 연계 동선 구축
- 분양 관련 테이블과의 데이터 연동 로직 작성

---

## 사용 기술

- Spring MVC
- JSP, JSTL
- JdbcTemplate
- Oracle
- Maven, Git

---

## 구현 시 중점 둔 내용

- 단순 게시글이 아니라, 상태가 변하는 프로세스로 기능을 설계했습니다.
- 분양 리스트와 상세 화면에서 동일한 상태 기준이 적용돼 일관성 있게 정보를 볼 수 있도록 하였습니다.
- 승인과 완료 처리 시, 단순히 화면에서만 막지 않고 서버 단에서도 한 번 더 검증해 안전성을 높였습니다.
- 완료 후에는 후기 작성과 알림까지 자연스럽게 이어져, 사용자가 다음 행동을 바로 찾을 수 있도록 플로우를 연결했습니다.
